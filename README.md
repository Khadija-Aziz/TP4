Exécution de App.java:


Cette exécution montre le fonctionnement de Hibernate avec trois stratégies d’héritage : SINGLE_TABLE pour les véhicules, JOINED pour les employés et TABLE_PER_CLASS pour les produits.

Hibernate crée les tables dans la base H2 en mémoire, insère les données (voitures, motos, développeurs, managers, livres et produits électroniques), puis les récupère correctement avec des requêtes SQL.

À la fin, toutes les tables sont supprimées automatiquement et le programme se termine avec exit code 0, donc sans erreur
